# Checkbox

<Description>A control that allows the user to toggle between checked and not checked.</Description>

<ComponentPreview name="checkbox" file="checkbox-demo" />

## Features

<FeatureList items={[
    'Full keyboard navigation.',
    'Supports indeterminate \_state.',
    'Can be controlled or uncontrolled.'
]} />

## Anatomy

```html
<div rdxCheckboxRoot>
    <button rdxCheckboxButton>
      <lucide-angular rdxCheckboxIndicator name="check" />
    </button>
    <input rdxCheckboxInput />
</div>
```

### Button
`RdxCheckboxButtonDirective`

It mirrors the checkbox state via ARIA/data attributes, toggles on click, and prevents Enter activation per WAI-ARIA.
In forms, it stops the button's click from bubbling so only the hidden input emits the native event used for form/validator integration.

<DataAttributesTable attributes={[
    { name: '[data-state]', value: '"checked" | "unchecked" | "indeterminate"' },
    { name: '[data-disabled]', value: 'Present when disabled' }
]} />


### Indicator
`RdxCheckboxIndicatorDirective`

Renders when the checkbox is in a checked or indeterminate state.
You can style this element directly, or you can use it as a wrapper to put an icon into, or both.

<DataAttributesTable attributes={[
    { name: '[data-state]', value: '"checked" | "unchecked" | "indeterminate"' },
    { name: '[data-disabled]', value: 'Present when disabled' }
]} />

### Input
`RdxCheckboxInputDirective`

It keeps form-related attributes in sync and dispatches native click events when the "checked" attribute changes so forms,
validators, and external libraries receive updates.

## Accessibility

Adheres to the [tri-\_state Checkbox WAI-ARIA design pattern](https://www.w3.org/WAI/ARIA/apg/patterns/checkbox).

## Managing Checked State

This section covers how to manage the `checked` state of the Checkbox.

```ts
@Component({
    template: `
        <button (click)="myChecked.set(true)"> Uncheck </button>

        <div rdxCheckboxRoot [checked]="myChecked()">
          ...
        </div>
    `
})
export class MyCheckbox {
    myChecked = signal(true);
}
```
