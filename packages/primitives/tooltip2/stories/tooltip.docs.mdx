# Tooltip2

#### Displays contextual information when a trigger is hovered or focused.

Tooltip2 composes Popper, Portal, Presence, and Dismissable Layer. It manages visibility and pointer grace areas while
leaving visual styling to the consumer.

<Canvas of={Stories.Default} />
<Controls />

## Anatomy

```html
<ng-container rdxTooltip>
    <button rdxTooltipTrigger>+</button>

    <div rdxTooltipPortal [container]="tooltipContent">
        <ng-template rdxTooltipPortalPresence #tooltipContent>
            <div rdxTooltipContentWrapper>
                <div rdxTooltipContent>Add to library</div>
                <span rdxTooltipArrow></span>
            </div>
        </ng-template>
    </div>
</ng-container>
```

`rdxTooltipContentWrapper` owns positioning. Apply visual styles and read `data-side` or `data-align` from the inner
`rdxTooltipContent` element.

## Moving Anchors

Tooltip2 uses `updatePositionStrategy="always"` by default so that it can follow moving triggers. Keep that strategy
when the trigger moves continuously through a CSS transform, animation, pointer tracking, or slider dragging.

<Canvas sourceState="hidden" of={Stories.Slider} />

```html
<div updatePositionStrategy="always" rdxTooltipContentWrapper>
    <div rdxTooltipContent>Content that follows a moving trigger</div>
</div>
```
